Funktione

doQumentation määt us IBM Quantum singem Open-Source-Inhalt en interaktive Lännplattform met live Code-Ußföhrung, Simulatormodus un Lännfortschretts-Verfoljung.

Inhaltsbibliothek

381 Sigge met Inhalt

42 Tutorials, 171 Anleitunge, 154 Kurssigge un 14 Module — all vun IBM Quantum singe Open-Source Qiskit-Dokumentazion.

Tutorials aanluure →

Auto-Synk us em Upstream

Dä Inhalt weed automatisch vun IBM singem GitHub-Repositorium synkroniseet, sulang blieve de Tutorials un Kurse aktuell met de neieste Qiskit-Veröffentlechunge.

Struktureete Navigazion

Automatisch jenereeete Siggelisse met zoklapppare Kattejorije. Tutorials, Anleitunge, Kurse un Module han jede en eijene organiseetde Siggeliss.

Live Code-Ußföhrung

Run / Back Toggle

Jede Notebook-Sigg hät ene Run-Knopf, dä all Code-Zelle met enem live Jupyter-Kernel aktiveet. Kleck Back för zrogg zor statische Aansicht ze jonn.

Drei Ußföhrungs-Backends

Kostenlose Fännußföhrung övver Binder (kein Installation nüdich), voll offline övver Docker, oder selbss jehoss op enem RasQberry Pi.

Backend konfigureeere →

IBM Cloud Code Engine

Serverless Jupyter kernel on your IBM Cloud account. Cold start in seconds instead of Binder's 10–25 minutes. Free tier covers roughly 14 hours per month.

Set up Code Engine →

Workshop Mode

Run classroom workshops with up to 50 participants. Deploy multiple Code Engine instances and share a single URL — participants are automatically distributed with sticky sessions and automatic failover.

Workshop setup guide →

Workshop Monitoring

Live dashboard with auto-refresh (30s) shows per-instance kernel counts, busy kernels, connections, memory usage, and uptime. Each instance exposes a /stats endpoint. Stress test script simulates concurrent users to find capacity limits before the class starts.

Settings dashboard →

Open in Google Colab

Every notebook page has an 'Open in Colab' button for zero-install cloud execution. Opens translated notebooks when browsing in another language.

Zelle-Ußföhrungs-Feedback

Visuell Feedback för jede Zell: amber Rand beim Laufe, jrön wann fäädich, root bei Fähler. En Legende erschingk en de Toolbar, wann dä Kernel paraat es.

Kontextbezoje Fählerhendeise

Automatische Erkennung vun ModuleNotFoundError, NameError un Kernel-Verbindungsavbroch. Akzionieerbare Hendeise erschinge unger dä Zell met vorjeschlaane Lössunge.

Ein-Kleck pip Install

Wann en fehlendes Paket erkannt weed, erschingk ene kleckbare 'Install'-Knopf. Noh dä Installation läuf de fehlgeschlaane Zell automatisch neu.

En JupyterLab opmaache

Jede Notebook-Sigg hät ene Knopf för de vullständije .ipynb-Datei en JupyterLab ze öffne för erweiderte Beärbeitung un Erforschung.

Run All & Restart

Run All executes every cell in order with a live progress counter. Stop after the current cell anytime. Restart gives you a fresh kernel without losing your Binder session.

IBM Quantum-Integratzion

Zugangsdaate-Speicher

Spächere Ühr IBM Quantum API-Token un CRN einmol en de Instellunge. Weed lokal en Ührem Browser jespeichert met 7-Daach Auto-Avlaaf för Secherheit.

Zugangsdaate inrichte →

Auto-Injekzion

Zugangsdaate weede beim Start automatisch en dä Kernel injizeet. Kein Nood, Tokens en jedes Notebook ze plakke — einfach Run klicke.

Simulatormodus

All Notebooks ohne IBM Quantum-Konto laufe losse. Wählt AerSimulator för ideale Simulazion us, oder peck us dutzende FakeBackends, de echte Jeräterausche simuleeere. Null Instellung erforderlich.

Simulator aktiveeere →

Ußföhrungs-Modus-Badge

De Toolbar zeich, welche Modus aktiv es — dä Simulatorname (z.B. 'FakeSherbrooke') oder 'IBM Quantum' — su dat Ehr immer wess, wie Ühr Code läuf.

Länne & Fortschrett

Fortschretts-Verfoljung

Sigge, die Ehr besöök, krieje en Häkche (✓) en de Siggeliss. Notebooks, die Ehr ußföhrt, krieje en Afspeelsymbol (▶). Verfollt Ühr Reise dörrch dä Inhalt.

Kattejorie-Badges

Jede Siggeliss-Kattejorie zeich en Badge wie '3/10', su dat Ehr dä Fortschrett op enem Bleck sieht. Kleck för Fortschrett pro Sekzion ze lössche.
